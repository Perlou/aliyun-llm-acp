2. 在大模型提示工程中，“要求模型先分析问题类型，再拆解解决步骤，最后输出结论” 的提示方式，对应的核心技术是？

A. 少样本提示

B. 思维链（Chain-of-Thought）提示

C. 角色设定提示

D. 指令优化提示

答案：B. 思维链（Chain-of-Thought）提示 是正确答案。大纲 “提示工程核心方法” 明确指出，思维链提示的核心是引导模型分步推理，通过 “分析 - 拆解 - 结论” 的逻辑链条提升复杂问题解答准确率，与题干描述完全匹配；其他选项中，少样本提示侧重提供示例，角色设定侧重赋予模型身份，指令优化侧重语言表述优化，均不涉及分步推理。

3. 某企业开发内部知识库问答系统，需让大模型能精准回答企业内部规章制度、产品参数等涉密信息，且要求数据不对外泄露。以下哪种方案符合要求？

A. 使用公有云大模型 API，将涉密信息直接嵌入提示词

B. 基于开源大模型进行私有化部署，结合 RAG 接入内部知识库

C. 将涉密信息脱敏后，调用第三方大模型 API

D. 选用免费的通用大模型，仅输入非核心涉密信息

答案：B. 基于开源大模型进行私有化部署，结合 RAG 接入内部知识库 是正确答案。根据大纲 “大模型安全与隐私保护” 及 “RAG 技术应用” 考点，私有化部署可避免数据外泄，RAG 技术能让模型精准检索内部涉密知识库，同时不对外暴露原始数据；公有云 API、第三方模型存在数据泄露风险，脱敏或仅输入非核心信息无法满足 “精准回答涉密信息” 的需求。

4. 开发者在优化 RAG 应用时，发现检索到的文本段与用户问题相关性较低，导致模型回答准确率下降。以下哪种优化措施最有效？

A. 降低检索的相似度阈值

B. 减少向量数据库中的文本段数量

C. 优化文本分块策略，采用更小的分块粒度并保留上下文信息

D. 增加模型的推理步数

答案：C. 优化文本分块策略，采用更小的分块粒度并保留上下文信息 是正确答案。大纲 “RAG 技术优化” 明确说明，文本分块质量直接影响检索相关性，过小的分块会丢失上下文，过大的分块包含冗余信息，合理的小粒度分块 + 上下文保留能提升检索精准度；降低相似度阈值会引入更多无关文本，减少文本段数量会降低召回率，增加推理步数与检索相关性无关。

5. 以下关于大模型 “上下文窗口” 的描述，正确的是？

A. 上下文窗口越大，模型的推理速度越快

B. 上下文窗口的大小决定了模型能处理的最大输入文本长度

C. 所有大模型的上下文窗口大小都是固定不可调整的

D. 上下文窗口仅影响模型的文本生成能力，不影响理解能力

答案：B. 上下文窗口的大小决定了模型能处理的最大输入文本长度 是正确答案。根据大纲 “大模型核心特性” 考点，上下文窗口（Context Window）的核心定义是模型能接收和处理的最大文本长度（包括输入 + 输出）；A 错误，窗口越大通常推理速度越慢；C 错误，部分模型支持动态调整或通过技术手段扩展；D 错误，窗口大小直接影响模型对长文本的理解能力（如长文档总结、多轮对话连贯性）。

6. 某开发者使用阿里云 PAI 平台开发大模型应用，需将模型部署为 RESTful API 供前端调用，且要求支持高并发请求。以下哪种部署方式最适合？

A. 单机部署模型，直接暴露 HTTP 接口

B. 使用 PAI-EAS（弹性计算服务）进行容器化部署，配置自动扩缩容

C. 将模型部署在本地服务器，通过内网穿透供前端调用

D. 采用静态文件部署方式，将模型权重文件直接提供给前端

答案：B. 使用 PAI-EAS（弹性计算服务）进行容器化部署，配置自动扩缩容 是正确答案。大纲 “阿里云大模型平台应用” 明确介绍，PAI-EAS 支持容器化部署大模型，自动扩缩容功能可应对高并发请求，且提供稳定的 RESTful API 接口；单机部署、内网穿透无法支撑高并发，静态文件部署无法实现模型推理功能。

7. 在大模型多轮对话应用开发中，为避免模型遗忘前文信息，以下哪种做法最有效？

A. 每轮对话仅向模型输入当前用户问题

B. 将历史对话记录与当前问题一起作为输入传递给模型

C. 要求模型每轮对话后总结前文要点并保存

D. 减少每轮对话的输出文本长度

答案：B. 将历史对话记录与当前问题一起作为输入传递给模型 是正确答案。根据大纲 “多轮对话系统开发” 考点，大模型不具备长期记忆能力，需通过 “历史对话 + 当前问题” 的输入方式，让模型在上下文窗口中获取前文信息，确保对话连贯性；A 会导致模型遗忘前文，C 增加模型负担且可能遗漏关键信息，D 与记忆前文无关。

8. 某公司开发一款智能文案生成工具，要求模型生成的文案符合品牌调性（如专业、简洁），且避免生成重复内容。以下哪种提示策略最有效？

A. 仅在首次调用时说明品牌调性，后续不再提及

B. 在每次提示中明确品牌调性要求，并添加 “禁止重复生成相同表述” 的约束

C. 提供 100 条符合品牌调性的文案作为示例，让模型学习

D. 选用参数更大的模型，依赖其自身能力生成符合要求的文案

答案：B. 在每次提示中明确品牌调性要求，并添加 “禁止重复生成相同表述” 的约束 是正确答案。大纲 “提示工程优化策略” 指出，明确且持续的指令约束能确保模型输出一致性，针对重复问题需直接添加禁止性约束；A 会导致模型后续输出偏离调性，C 示例数量过多可能超出上下文窗口，且效率低，D 参数大小与是否符合品牌调性、是否重复无直接关联。

9. 关于 RAG（检索增强生成）技术的核心作用，以下描述最准确的是？

A. 提升大模型的训练效率，减少训练数据量

B. 扩展大模型的知识边界，解决 “知识过时” 和 “领域知识缺失” 问题

C. 降低大模型的推理延迟，提升响应速度

D. 增强大模型的多模态处理能力（如文本、图像、语音）

答案：B. 扩展大模型的知识边界，解决 “知识过时” 和 “领域知识缺失” 问题 是正确答案。大纲 “RAG 技术核心原理” 明确，RAG 通过检索外部知识库的最新 / 领域特定数据，补充模型训练数据之外的信息，从而解决模型知识固化（过时）和领域适配不足的问题；A 错误，RAG 不涉及模型训练；C 错误，RAG 可能增加检索环节的延迟；D 错误，RAG 核心聚焦文本知识补充，不直接增强多模态能力。

10. 开发者在使用大模型进行文本分类任务时，发现模型对小众类别（如 “投诉 - 物流延误”）的识别准确率极低。以下哪种优化方案最适配？

D. 简化分类任务，合并小众类别

答案：B. 采用少样本提示，提供小众类别的多个示例 是正确答案。根据大纲 “大模型微调与提示优化” 考点，针对小众类别数据稀缺的问题，少样本提示（Few-shot Prompting）通过提供少量高质量示例，能快速引导模型识别小众类别；A 增加所有类别数据会浪费资源，且小众类别占比仍低；C 参数大小不直接解决类别识别问题；D 合并类别会损失任务精度，不符合需求。

12. 在大模型应用开发中，“模型微调（Fine-tuning）” 的核心目的是？

A. 降低模型的推理延迟

B. 让模型适配特定领域或任务的需求

C. 增加模型的上下文窗口大小

D. 减少模型的参数数量

答案：B. 让模型适配特定领域或任务的需求 是正确答案。根据大纲 “大模型微调技术” 考点，微调是通过在特定领域数据上进一步训练，使模型学习领域知识或任务范式，提升特定场景的性能；A 错误，微调可能增加模型复杂度，延迟升高；C 错误，上下文窗口大小由模型架构决定，微调无法改变；D 错误，微调不减少参数数量，部分微调方法（如 LoRA）仅新增少量参数。

16. 在大模型提示词设计中，“明确任务目标、限定输出格式、提供核心约束条件” 属于哪种提示原则？
